# Trending Diggest

## 产品愿景

<!-- 一句话：为谁解决什么问题，产生什么价值 -->
为中文技术读者，把分散的公开技术资源（当前实现：Claude 官方博客）自动抓取、生成中文双语译读、归档为按时间组织、带索引可检索的 markdown 库，降低跟读外文技术更新的门槛与重复劳动。

**目标用户**

| 用户类型 | 核心痛点 | 期望成果 |
|---------|---------|---------|
| 中文技术读者 | 外文博客更新分散、英文阅读慢、易漏 | 增量、及时的中文译读归档，一处可检索 |
| 团队/下游消费方 | 需要结构化、可引用的内容源 | 稳定的 markdown 归档 + index 供站点/钉钉文档消费 |

---

## 产品范围

**在范围内**
- 多源抓取架构（fetch 原文 → raw 归档），claude-blog 为首个 source 实现
- 增量抓取 + 幂等去重（state/processed.json，已处理 URL 不重复）
- 中文双语译读 markdown 生成
- 按 年/月 组织的归档目录 + index.md 索引表维护
- 仅归档公开可访问内容（合规边界）

**明确排除**
- 抓取/译读/调度的**自动化脚本与凭据** —— 归私有 coworkspace 仓（本仓只承接产出）
- 面向用户的 **Web 展示前端**（trending.theuntold.io）—— **当前**归 `projects/external/theuntold`（独立 SDLC）。⚠️ 该前端迁入本仓的方案正在 theuntold 项目讨论中，未定论；已登记为 Backlog 条目 `frontend-migration-from-theuntold`（阻塞待结论），定论后再纳入本仓 specs（届时新增 `ui/`）
- 非公开 / 需登录才能访问的源内容

---

## 约束

<!-- 产品级硬约束：不可以做什么、必须兼容什么 -->

- 公开仓库：只允许存放公开可访问的 markdown 归档与原文链接，不得纳入受限内容或凭据
- 增量幂等：重复运行不得产生重复归档或破坏既有产物
- 归档产物是下游（theuntold 站点、钉钉文档）的数据源，目录结构与命名变更属契约级改动
