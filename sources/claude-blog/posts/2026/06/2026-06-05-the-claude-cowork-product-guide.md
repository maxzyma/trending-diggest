# Claude Cowork 产品指南

> 来源：Claude Blog，2026-06-05
> 原文链接：https://claude.com/blog/the-claude-cowork-product-guide
> 分类：Enterprise AI

## 核心要点

- Claude Cowork 是一款新的知识工作代理（knowledge work agent），运行在 Claude 桌面应用中，可将工作从对话延伸到真实的交付成果。
- 它能读写本地文件、跨 Slack 和 Google Drive 等已连接应用协作，并将多步骤任务执行到底，同时附带回溯到原始文件与消息的引用（citations）。
- 用户描述目标、期望成果与节奏（cadence），Claude 会规划步骤并完成工作，确保用户全程参与。
- 产品矩阵明确了工具的使用场景：聊天用于对话式起草、Claude Code 用于编程、Claude Cowork 用于跨应用的知识工作。
- 指南涵盖核心能力（本地文件访问、子代理、长时运行任务、定时任务）、安装与系统要求、权限模型，以及上手建议。
- 提供七种常见 Claude Cowork 工作流，包括研究简报、会议准备和周期性报告。
- 可使用插件（plugins）来支持营销、产品管理等特定知识工作。

## 中文译文

大多数 AI 工具都是对话式的。你提出一个问题，得到一个答案，而把这个答案转化为真正有用的东西——一份演示文稿、一篇文档、一张电子表格、一封邮件——仍然需要手动完成。

Claude Cowork 是我们推出的新知识工作代理（knowledge work agent），它让你能够把这些工作委托给 Claude，从而把精力集中在那些占据你时间的、更具战略性和创造性的问题上。Claude Cowork 运行在 Claude 桌面应用中，能够读写本地文件、跨 Slack 和 Google Drive 等已连接应用协作，并将多步骤任务执行到底，形成真实的交付成果，同时附带回溯到实际文件与消息的引用（citations）。你描述目标、期望成果与执行节奏（cadence），Claude 便会规划步骤并完成工作，确保你在每一步都参与其中。

为帮助你快速上手，我们整理了一份实用的 Claude Cowork 产品指南。我们分享了以下内容：

- Claude Cowork 与对话式 AI 工具有何不同，以及它的核心能力，包括本地文件访问、子代理（subagents）、长时运行工作（long-running work）和定时任务（scheduled tasks）

- 一份产品矩阵，用于理解何时使用何种工具：聊天用于对话式起草、Claude Code 用于编程、Claude Cowork 用于跨应用的知识工作

- 安装与系统要求、权限模型（permissions model），以及值得先尝试的内容

- 七种常见的 Claude Cowork 工作流，包括研究简报、会议准备和周期性报告

- 如何使用插件（plugins）来处理营销和产品管理等特定知识工作
