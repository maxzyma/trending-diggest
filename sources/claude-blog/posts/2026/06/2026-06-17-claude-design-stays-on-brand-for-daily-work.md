# Claude Design 现在能在日常工作中保持品牌一致

> 来源：Claude Blog，2026-06-17
> 原文链接：https://claude.com/blog/claude-design-stays-on-brand-for-daily-work
> 分类：Product announcements

## 导语

Claude Design 现在能在各个项目中坚持你的设计系统（design system），与 Claude Code 流畅协作，让你直接在画布（canvas）上编辑，并连接到更多你已经在使用的工具。它在 Claude 桌面应用的侧边栏中有了新的入口；你也可以在 claude.ai/design 找到它。

## 核心要点

- Claude Design 现在能在项目间始终遵循你的设计系统，并在你看到结果之前先对照设计系统进行修正。
- 与 Claude Code 的双向同步更顺畅：用 `/design-sync` 拉取设计系统，用 `/design` 在终端内创建、编辑和同步设计项目。
- 全新编辑器提供对每个元素的精细控制，新增丰富的布局控制，并修复了数百项稳定性问题。
- Claude Design 现已与聊天、Claude Cowork 和 Claude Code 共享使用额度，平均每次交互消耗更少的令牌（token），错误率大幅下降。
- 可可靠导出为 PDF 和 PowerPoint，连接器现已支持 Adobe、Base44、Canva、Gamma、Lovable、Miro、Replit、Vercel 和 Wix。
- 上线首周已有超过一百万人使用 Claude Design。

## 中文译文

![Image 1](https://cdn.prod.website-files.com/68a44d4040f98a4adf2207b6/6a32d4f7ae9712d1f8ef661e_Claude-Design-beta-static.webp)

Claude Design 现在能在各个项目中坚持你的设计系统（design system），与 Claude Code 流畅协作，让你直接在画布（canvas）上编辑，并连接到更多你已经在使用的工具。它在 Claude 桌面应用的侧边栏中有了新的入口；你也可以在 claude.ai/design 找到它。

我们的目标与此前一致：让你探索比截止期限通常允许的更多方向，并让交付给精修或开发环节的过程尽可能无缝。如今这个目标服务于更多人。Claude Design 上线首周就有超过一百万人使用，你的反馈正在塑造我们接下来的开发方向。

### 使用你的设计系统组件进行构建

我们重建了设计系统导入功能，让它更灵活、更精确。你可以从 GitHub 仓库、设计文件或原始上传中引入一个或多个设计系统。Claude 使用你的组件（component）进行构建，对照你的设计系统检查其输出，并在你看到之前完成修正。对于较大的团队，新增的管理员（admin）角色可以批准一套标准系统并锁定编辑，从而让成果始终符合你公司的规范。

### 在设计与代码之间切换

现在在 Claude Design 与 Claude Code 之间切换工作、同时保持工作同步变得更容易，相关更新于今日推出。使用 `/design-sync` 拉取你的设计系统，这样你在 Claude Design 中构建的一切都从你已有的组件出发。当一项设计准备好变成软件时，你可以将它交付给 Claude Code，它会从你已有的工作继续，而不是从一张截图重新开始。

更喜欢从 Claude Code 开始？你可以使用 `/design` 在不离开终端的情况下创建、编辑和同步设计项目。把设计导入你的代码库，把你的代码变成实时原型（live prototype），或者让 Claude 把一个项目从头到尾完成。

> "Claude Design 让我在拥有前沿模型智能的同时，也拥有久经考验的设计工具的功能与能力。无论我在为 Tenex 的网站设计方向、新的品牌资产还是演示文稿工作，Claude Design 都是我去的第一个地方。平易近人的用户体验与出色的品味及设计直觉相结合，正是它成为我技术栈核心部分的原因。而 Claude Design 与 Claude Code 之间的交付，让从原型到生产的过程变得无缝。" —— Alex Lieberman，Morning Brew 与 Tenex 联合创始人

### 在日常工作中更稳定

我们的新编辑器让你对设计中的每个元素都拥有直接、精细的控制。全新、丰富的布局控制让你拖动、调整大小和对齐元素。数百项稳定性修复让编辑器在真实使用中更经得起考验。

Claude Design 现在也与聊天、Claude Cowork 和 Claude Code 共享使用额度，因此大多数人将获得更多的创作余量，而那些触及上限的人触及频率会更低。如今平均每次交互使用更少的令牌（token）就能达到同样的效果，错误率也大幅下降。

### 从画布到你使用的工具

可靠地导出为 PDF 和 PowerPoint，或将你的作品发送到你已经在使用的应用——连接器（connector）列表现已包括 Adobe、Base44、Canva、Gamma、Lovable、Miro、Replit、Vercel 和 Wix，更多目标平台即将到来。
