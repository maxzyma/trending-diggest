# 来认识 Built with Opus 4.7 Claude Code 黑客松的获奖者

> 来源：Claude Blog，2026-06-15
> 原文链接：https://claude.com/blog/meet-the-winners-of-built-with-opus-4-7-claude-code-hackathon
> 分类：Claude Code

## 导语

从医学培训、电子设备维修，到编程教育和工厂维护，看看我们最近这场线上黑客松获奖者们打造的项目。

## 核心要点

- 一等奖 Medkit 是一款面向住院医师和初级医生的学习工具，在游戏化的医疗诊所中模拟真实的患者接诊场景。
- 二等奖 Wrench Board 借助 Opus 4.7 理解可视化电路图的新能力，帮助独立技术人员排查复杂的电子维修问题。
- 三等奖 Maieutic 是一款引导学生在关键时刻放慢节奏、先讲清思路再写代码的 IDE，帮助学生闭合元认知循环。
- 「最具创意 Opus 4.7 应用奖」由 Virtual Puppet Theater 获得，将摄像头视频和语音转换为动态交互式木偶剧。
- 「保持思考奖」颁给 MaestrIA，它为普通人提供大师级的家庭维修诊断，同时让熟练匠人有渠道展示专业能力。
- 「最佳 Claude Managed Agents 应用奖」由 ARIA 获得，它持续监测工厂机器并在故障出现时即时生成诊断和维修方案。
- 多位获奖者强调「先做评估、先做规划」以及把 Claude 当作思考伙伴的重要性。

## 中文译文

上周，我们举办了 Claude Build Day，这是我们最新的一场黑客松，建造者们齐聚旧金山，使用 Claude Opus 4.8 把想法付诸实践。

在我们等待看到他们的作品的同时，我们与 Built with Opus 4.7 黑客松的获奖者们聊了聊他们的项目。他们的项目涉及医学培训、电子维修、计算机科学教育、互动游玩、家庭维修和工厂维护。

恭喜各位获奖者以及所有参与者！我们希望他们的想法能给你带来启发。

### 一等奖：Medkit，Bedirhan Keskin

来自伊斯坦布尔、从医生转型为软件工程师的 Bedirhan Keskin，使用 Claude Managed Agents 打造了 Medkit：一款面向住院医师或初级医生的学习工具，在游戏化的医疗诊所中模拟真实的患者接诊场景。

「当你独自一人待在急诊科，有 50 名患者在等候，而你意识到有些病例你在医学院里从未实操过时，你最终就会在真实患者身上实时练习这些病例，」Bedirhan 说。

借助 Medkit，医学生可以练习诊断和治疗模拟患者，使用这个工具采集病史、开化验单、读影像、做诊断并开具治疗方案。在接诊结束时，一个智能体评分器会依据与考试委员会考官所用相同的、已公开发布的临床指南，对整个接诊过程进行评估。

Bedirhan 在四个独立的 Claude Code 会话（语音引擎、内容生成、3D 游戏层和核心应用）中构建了 Medkit，让每个上下文都保持干净，并同时推进所有部分。他遵循「说，而不是打字」（talk, don't type）的方式，几乎完全通过语音工作。

Medkit 已经获得关注，三所医学院和一家制药公司（均位于伊斯坦布尔）将在未来几周内开始试点。

给其他建造者的建议：把 Claude 当作思考伙伴，而不仅仅是编码智能体。

Bedirhan 最初的本能是自托管语音引擎，但 Claude 建议使用云服务商以加快进度。「我最看重 Claude 的一点是，它不只是一个代码生成器，而是一个思考伙伴，帮我看到那些原本会错过的选项，」他说。

### 二等奖：Wrench Board，Alexis Chapellier

来自法国 Reignier-Ésery 的 Alexis Chapellier，在创建 RepairMind（一个面向维修店的 AI 驱动管理平台）之前，曾从事多年电子维修工作。他的 Opus 4.7 黑客松项目 Wrench Board，帮助独立技术人员搞定复杂的维修。用户放入一张电路图和一张板视图（boardview）并描述故障现象，智能体会创建一张统一的电气图，对其进行推理，指出确切需要探测的焊盘，读取测量数据，并不断更新假设直到诊断出问题。

Alexis 在 Claude Design 中为 Wrench Board 做了原型，把应用的各项职责（设计、电路图导入、板视图、诊断智能体）分离开，先为每一项生成规格（spec），然后再生成计划。他在 Claude Code 的多智能体模式下执行，在调试时并行运行五六个智能体、每个领域配一个专属智能体，在每一步进行基准测试。

Alexis 的大赌注押在 Opus 4.7 理解可视化电路图的新能力上；他说，当他要求模型追踪主板上一条供电路径时，他就知道这套方案行得通了。

「我看着 Wrench Board 的板视图一步步亮起来，箭头出现，元件被指了出来，名称浮现。在那一刻，我明白了这个想法站得住脚，」他说。

Wrench Board 的下一阶段是建立一个由有兴趣试用这款应用的电子维修人员、以及能够用现场经验丰富这个工具的专家组成的社区。他的 Claude 额度将用于 RepairMind、这些首批用户，以及当前正在推进的所有基础设施。

「这场黑客松证明了，一个从维修店走出来的自学者可以在五天内交付一个雄心勃勃的系统，」Alexis 说——他参赛时还正在申请「糊口的工作」。「Claude Code 会放大任何拥有想法、并有毅力去执行的人，无论他的起点在哪里。」

给其他建造者的建议：在头脑风暴中深入，并对模型提出反驳。

Alexis 使用 Superpowers，这是一个集成在 Claude 中的技能框架，它把「先头脑风暴、再做计划」的步骤结构化，有时还会并行运行多个头脑风暴，以在不同方向上同时推进。他从 Claude Design 开始，然后用内置按钮直接共享项目，交接到 Claude Code，并在模型说「不行」时推它一把。

「黑客松期间，Claude 好几次告诉我，这个或那个在可用时间内做不完。但实际上，我有充裕的时间，」他说。「你得知道怎么对它说：我无论如何都要试试看。」

### 三等奖：Maieutic，Paula Vásquez-Henríquez

在智利康塞普西翁的 Universidad del Desarrollo 教授计算机科学的 Paula Vásquez-Henríquez 说，过去两年她看到越来越多的学生在不理解自己代码的情况下通过考试。

「学生现在用 AI 来制造代码，但他们根本不知道代码在做什么，」她说。「他们从没学会精确地陈述一个问题，没学会在编码前先草拟一个计划，也没学会批判性地阅读自己的代码、发现它在哪里偏离了原本的意图。自动补全在他们还没把问题想清楚之前就交出了能运行的代码，于是那个元认知循环——那种真正造就程序员的『对思考进行思考』——永远闭合不了。他们毕业时能生成代码，却无法对代码进行推理。」

Paula 目前正在攻读人工智能方向的博士学位，研究学生与 AI 的交互模式，她参加这场黑客松是为了从学生和教师两个视角来解决这个问题。

Maieutic 是一款旨在让学生在关键时刻放慢节奏的 IDE。学生必须先用平白的语言描述他们的程序应该做什么，然后才能写任何代码；Claude 会提出有针对性的澄清问题，并让编辑器保持锁定，直到规格足够详细，详细到一个有能力的程序员无需猜测就能实现它为止。

之后学生可以开始写 Python，但自动补全是关闭的；一个聊天面板会直接回答查询性问题，但对推理性问题则用反问而不是直接修复来回应，拒绝替学生思考。

工具的核心是「意图—差异审查」（Intent-Diff Review），它让 Claude 把规格与最终代码进行对比，把每一处分歧归类为偏移（drift）、修订（revision）或缺陷（bug），然后抛出一个中立、不带指责的问题，促使学生自己解释这个问题。

对于教师，一个实时仪表盘会为每个学生显示一行，附带一句话的认知摘要（例如「已写了三次规格，仍未考虑空输入」）。教师可以点击单个学生，监测他们与 Claude 的具体交互；Claude 还会分析整个班级，识别并呈现全班共有的误解，让教师能够弥合这一差距。

黑客松结束以来，休斯顿大学的研究人员已就合著论文一事与她联系，Paula 正把她的奖金额度投入到进一步开发这个工具上。她说，黑客松这一周让她看到，理解一个问题与为它交付一个工具之间的鸿沟已经坍塌了。

「我是智利康塞普西翁的一名教育工作者，不是硅谷的，」她说。「我在一周内交付了一个可用的全栈产品，因为这些工具让我能够留在我真正擅长的角色里，而它们处理其余的事情。最贴近真实问题的人，现在可以直接为这些问题来构建解决方案了。」

给其他建造者的建议：先思考，再构建。

这个项目是 Paula 在实践她自己的理念：先写规格，再构建。「Maieutic 之所以存在，是因为学生直接跳到代码，而我把它做好的唯一办法，就是拒绝让自己也这么做，」她说。她用了整整两天进行纯粹的思考工作，在写下一行代码之前先创建了设计规格和技术规格。

「那两天的规格工作在当时感觉很慢，黑客松里有种立刻开始交付的真实压力，但正是它们让这一周剩下的时间得以快速推进，」她说。

### 最具创意 Opus 4.7 应用奖：Virtual Puppet Theater，Rene Hangstrup Møller

被 Opus 4.7 的空间推理能力所吸引，全栈开发者 Rene Hangstrup Møller 打造了 Virtual Puppet Theater，这是一款基于浏览器的应用，它把摄像头视频和语音转换成一场动态的交互式木偶剧。一个实时动画木偶会镜像用户的动作，同时第二个由 AI 驱动的伙伴木偶会与用户斗嘴；口头指令可以即时改变场景并生成 3D 道具。

Rene 在整条流水线中都使用了 Claude：概念讨论、规划和代码编写，而他自己负责导演、架构、审查和决策。这个应用基于 Bun、Vite 和 TypeScript，使用 MediaPipe 手部追踪（运行于 WASM）和 Three.js 以 60 fps 在 3D 中渲染木偶舞台。一个小型 WebSocket 服务器通过 Anthropic SDK 连接到 Claude Opus 4.7，来驱动 AI 木偶的对话并即时生成 3D 道具，而语音方面输入由 Web Speech API 处理、输出由 ElevenLabs 处理（以浏览器语音合成作为后备）。Opus 的空间推理能力通过基于截图的反馈循环不断打磨，负责处理视觉输出。

Virtual Puppet Theater 除了开放式的游玩之外没有任何目标，Rene 说他对这个获奖项目没有产品计划。对他而言，这关乎学习和乐趣。「我让我最小的儿子测试了它，他玩得不亦乐乎，」Rene 说。「看着他与木偶互动、描述场景、对回应咯咯发笑，这真的就是我所需要的全部用户验证了。」

他补充说，Virtual Puppet Theater 的源代码已在 GitHub 上以 MIT 许可证开放，「如果有人想把它做得更深入的话。」

给其他建造者的建议：如果你要参加黑客松，请安排时间制作演示视频。

「制作一个 3 分钟的视频所花的时间比你想象的要长得多，」Rene 说，并指出他在使用 Claude 和 Hyperframes 创建和剪辑他的 Virtual Puppet Theater 视频时差点赶不上黑客松的截止时间。「黑客松 Discord 里很多人都警告过这一点，他们说得对。下次我会把最后一整天专门留出来做演示。」

### 「保持思考」奖：MaestrIA，Benjamin Torralbo

Benjamin Torralbo 从小跟着他的父亲 Juan Rodrigo Torralbo 当学徒——他父亲是智利奇洛埃岛一位获得认证的 Maestro Mayor（木工大师）。「我父亲有 30 年的手艺，修复过联合国教科文组织名录上的教堂，但他在智利的体系里依然是隐形的，就像其他数十万名手艺人一样，」Benjamin 说。「与此同时，需要做家庭维修的人不知道问题出在哪、要花多少钱、该找谁，也不知道自己被收的费用是否公道。」

他的 MaestrIA 黑客松项目同时解决了两端的问题：作为一款 Web 应用，它为普通人提供大师级的家庭维修诊断，同时给熟练匠人提供一个展示专业能力的途径。

使用 MaestrIA 时，用户拍下他们的问题，用语音或文字描述它，并分享所在位置。Claude 会实时流式输出它的推理过程，在照片上叠加动画边界框，然后给出结构化诊断：哪里坏了、用什么材料、严重程度 1–5 级、项目预算和时间估算。随后智能体会渲染一张按行当筛选的附近匠人地图，同时第二个智能体起草一条可发送的 WhatsApp 消息。

MaestrIA 的技术核心是一个 JSON 文件，它被注入到每一次诊断中，包含 17 条诊断规则、7 种奇洛埃本地木材、16 个当地行当方言术语、19 个基准价格和 9 个手艺中的常见错误——这些全部提炼自 Benjamin 与他父亲进行的数小时访谈。在不触碰系统提示词（system prompt）的情况下，仅这一个文件就把他的评估分数提升了七个百分点（相对于一位人类大师的判断，从 74% 升到 81%），也正是它让 MaestrIA 能够诊断出「alerce 木板墙上的上升潮气」，而不是泛泛的「木材损坏」。

在没有任何编程经验的情况下，Benjamin 说他的角色是监督 Claude 技术执行的工地领班。「在编写任何功能之前，我都让 Claude Code 设计规格、分阶段的行动计划，以及安全模型：针对提示注入（prompt injection）的输入清洗、速率限制、来源验证，以及把 Zod 模式（schemas）作为唯一可信来源，」他说。「然后我逐个 diff 地审查每一个功能。」

Benjamin 希望 MaestrIA 能成长进入新建项目、五金店集成、正式预算、合同、评价和认证系统。最终，每个行当都会有它自己的 Maestro Mayor 被编码在里面，包括木工、建筑师、水管工、电工和泥瓦匠。

他的奖金额度用于开发这款应用、把他父亲的公司数字化作为一个实地试点，以及他自己的技术成长。「Claude Code 让一个来自奇洛埃、毫无编程经验的 20 岁年轻人，能够构建出他自己的父亲也能用的软件，而且这款软件还能帮助智利另外 28 万名像他一样的匠人，」他说。「它也为数百万一直拥有宝贵想法、却苦于无法将其变为现实的人打开了一扇门。」

给其他建造者的建议：先做评估，再做功能。

「我做的最重要的一件事，就是针对我父亲记录下真实标准答案的 12 个真实案例，构建了一个可审计的 9 维度评估，」Benjamin 说。「是那个评估，而不是我的直觉，告诉我什么有效、什么无效。如果我再参加一次黑客松，评估会是第一个提交（commit）。」

### 最佳 Claude Managed Agents 应用奖：ARIA，Idriss Benguezzou 和 Adam Hnaien

大多数工厂里都有那么一位资深技师，他光凭机器发出的声音就能判断它快要坏了。荣获最佳 Claude Managed Agents 应用奖的项目 ARIA（Adaptive Runtime Intelligence，自适应运行时智能）把一位经验丰富的维护工程师的直觉，变成了一套价格实惠、设置迅速的 AI 系统，它持续监测工厂机器，并在故障出现的那一刻生成定制化的诊断和维修方案。

使用 ARIA 时，维护工程师上传一份制造商的 PDF，回答四个平白语言的校准问题，15 分钟内整个工厂就被建好了画像。从那以后，五个智能体监测实时信号。如果某个智能体检测到一处故障或预测到故障即将发生，它就会生成一份工单，分析涉及的部件、故障模式、紧急程度、零件和干预窗口。

这个项目的两位建造者都有车间一线的工业经验，他们在黑客松寻找队友的 Discord 频道里相遇。Idriss Benguezzou 是一位拥有数据/AI 硕士学位的法国工业软件工程师，已经规划这个想法及其大部分架构有一段时间了。Adam Hnaien 是一位自学成才的工程专业学生，对 Claude Code 和多智能体工作流很有经验，他立刻认出 ARIA 是工业维护领域一个有价值的解决方案。

Idriss 和 Adam 把黑客松第二天的全部时间都花在规划模式上，配合一块 GitHub Project 看板，在写下第一行代码之前界定了每一个里程碑、议题和验收标准。「我们想从 M2 开始就以 200% 的状态投入，」Adam 说。「一天的规划让我们能把这一周剩下的时间用在执行上，而不是即兴发挥。」

两人都估计 Claude Code 写了约 80% 的原始代码行，而他们亲手做出领域逻辑和设计决策。Idriss 负责阈值评估、知识库（KB）模式和异常检测，因为他说，「你没法靠提示词就知道一个维护技师实际上在看什么。」Adam 负责用户体验、视觉语言和 ARIA 的星座（constellation）概念，因为他说，「你没法靠提示词凑出品味。」

Managed Agents 处理了智能体基础设施。「如果没有 Claude Managed Agents，我们这一周就得去构建 Anthropic 已经托管好的基础设施：一个沙箱化的 Python 环境、安全执行、会话持久化、MCP 分发，」Adam 说。「相反，我们把那一周用在围绕那套基础设施构建产品上。这就是用五天交付 ARIA 和用五周交付 ARIA 之间的区别。」

黑客松结果公布后，正在攻关这个完全相同问题的公司主动联系了这个项目。Idriss 会把 ARIA 的智能体架构、知识库模式和信号流水线整合进他自己的工业物联网平台；他的额度将用于更多的构建和实验。至于 Adam，他的计划是继续探索工业智能体 AI 中的机会，并用 API 额度继续构建和实验。
