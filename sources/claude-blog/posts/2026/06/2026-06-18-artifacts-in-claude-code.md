# Claude Code 现已支持工件（Artifacts）

> 来源：Claude Blog，2026-06-18
> 原文链接：https://claude.com/blog/artifacts-in-claude-code
> 分类：Claude Code

## 核心要点

- Claude Code 现可将工作进度捕获为工件（artifact），把会话中的工作转化为可实时更新、可共享的可视化页面。
- 工件基于会话的完整上下文构建，包括你的代码库、连接器（connectors）和对话本身，无需手动接入数据源或搭建基础设施。
- 页面会就地实时刷新，每次发布在同一链接生成新版本，并保留版本历史以便随时恢复。
- 调试是内部测试中最常见的用例之一，工件可让团队成员共享同一视图与上下文，减少状态同步沟通。
- 工件默认对作者私有，仅限组织内已认证成员查看，无法公开；管理员可进行访问控制、保留策略与合规管理。
- 当前以测试版（beta）形式向 Claude Team 与 Enterprise 组织开放。

## 中文译文

从今天起，Claude Code 可以将工作进度捕获为一个工件（artifact），把 Claude Code 的工作转化为实时、可共享的可视化页面——包括 PR 讲解、系统说明、仪表盘（dashboard）和发布检查清单——这些页面会随着你的会话推进而自我更新。

一次 Claude Code 会话的范围可以从排查事故到重构服务，再到分析数月的数据。工件把这些工作转译为任何人都能打开并浏览的网页，比如一份拉取请求（pull request）讲解、一个可筛选和排序的仪表盘，甚至是一份随工作完成而自动填写的发布检查清单。工件让协作共享工作变得更容易，团队可以把更多时间花在构建上，更少时间花在沟通状态更新上。

### 基于你会话的上下文构建

Claude Code 使用会话的完整上下文来构建工件，包括你的代码库、你的连接器（connectors）以及对话本身。一个单独的事故页面可以汇集来自代码的失败测试及其背后的函数、来自所连接监控工具的错误激增数据，以及来自你刚刚运行的会话的根因（root-cause）推理。有了工件，你无需接入数据源或搭建基础设施。你只需请求一个页面，Claude Code 就会从已有内容构建它。

### 就地更新的实时页面

当 Claude Code 更新一个工件时，已打开的页面会就地刷新，队友在更新发布的那一刻就能看到。每次发布都会在同一链接生成一个新版本，并带有版本历史，因此你可以随时恢复；同时一个图库（gallery）让你浏览和管理你所创建的所有工件。

在我们的内部测试中，最常见的用例之一是调试。这些场景通常是这样的：一名工程师在每日站会前启动一次事故调查。Claude Code 梳理日志并发布一个工件：一条时间线、可疑的提交（commits）以及一张错误率图表。她从页面顶部分享链接给团队。等到站会开始时，Claude 已经随着调查推进重新发布了两次，纳入了最新信息。有了工件，团队成员和利益相关方不必再让"智能体把发现的内容讲一遍"，因为他们看的都是同一个视图、同一份上下文。

### 仅限你的组织私有

每个工件默认对其作者私有。当你准备就绪时，可以直接从页面与你的队友和组织分享。工件只能由组织内已认证的成员查看，无法被设为公开。管理员通过组织级开关和基于角色的范围控制来管理访问权限，设置保留策略，并通过合规 API（compliance API）获得组织范围的可见性。

### 开始使用

向你的会话请求一个工件——或者直接请求一些可视化内容，以下是按角色划分的一些想法：

- 法务 / 开源：直接从代码仓库对每个依赖项进行许可证审计，标记出 copyleft（著佐权）。"构建一个工件，列出每一个第三方依赖项及其许可证，标记任何 copyleft 的项。"

- 隐私：在代码中描绘个人数据在哪里被收集、存储和记录的数据流图。"在整个代码库中追踪我们在哪里接触个人数据，整理为一个工件用于隐私审查。"

- 安全：将发现链接到精确的代码行，使修复明确无误。"将这次审查中的认证（auth）发现构建为一个工件，每一项都链接到代码。"

- 财务运营 / 平台财务（FinOps）：从你的基础设施即代码（infrastructure-as-code）映射云资源和成本驱动因素。"从 Terraform 中映射我们的云资源到一个工件，按服务分组，并标注主要成本驱动因素。"

- 软件工程师：从代码差异（diff）及其周边代码中提取的、审查者真正能跟上的 PR 或缺陷讲解。"做一个工件讲解这个 PR——差异、推理过程，以及我测试了什么。"

- 设计师与前端工程师：为一个界面提供多个用户体验（UX）方向，每个都基于你真实的组件构建，因此你选中的那个可以直接发布。"给我一个工件，包含这个注册表单的 5 种 UX 变体，基于我们的组件库构建。"

- 资深工程师与架构师：从真实的导入图（import graph）而非白板绘制出一个服务实际如何组合在一起的图谱。"从代码中将支付服务如何组合在一起映射为一个工件。"

- 站点可靠性工程师与值班人员（SRE & on-call）：一个随调查推进而扩展、并最终成为事后复盘（postmortem）的事故页面。"把这次事故变成一个工件——时间线、可疑提交、来自我们监控的错误激增——并在我推进工作时重新发布。"

- 工程经理：从已合并的 PR 构建出真正交付了什么的页面。"从 PR 中构建一个工件，展示本周我团队合并了什么，按项目分组。"

Claude Code 构建页面并给你一个链接。在你的浏览器或桌面应用中打开它，从页面顶部分享——更新会自动发布到同一个 URL。

### 可用性

工件以测试版（beta）形式向 Claude Team 和 Enterprise 组织开放，可从 Claude Code CLI 和桌面应用使用，页面可在任何浏览器中查看。

立即开始使用 Claude Code。
