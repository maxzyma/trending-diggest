# Claude 托管智能体新功能：做梦、结果导向与多智能体编排

> 来源：Claude Blog，2026-05-19
> 原文链接：https://claude.com/blog/new-in-claude-managed-agents
> 分类：Agents

## 核心要点

- 「做梦」（dreaming）作为研究预览版上线，它通过回顾历史会话发现模式，帮助智能体实现自我改进。
- 「结果导向」（outcomes）让你用评分标准（rubric）定义成功，由独立的评分器在单独上下文中评估输出并驱动智能体反复改进。
- 多智能体编排（multiagent orchestration）让主智能体把任务拆分并委派给各具模型、提示词与工具的专精子智能体并行处理。
- 「记忆」（memory）与「做梦」共同构成自我改进智能体的稳健记忆系统：记忆在工作中捕获所学，做梦在会话之间精炼记忆。
- 内部测试中，结果导向使任务成功率最多提升 10 分，在最难的问题上收益最大，并提升了文件生成质量。
- Harvey、Netflix、Spiral by Every、Wisedocs 等团队已用这些能力构建可自我核验、跨会话学习并并行处理复杂任务的智能体。

## 中文译文

今天，我们以研究预览（research preview）的形式在 Claude 托管智能体（Claude Managed Agents）中推出「做梦」（dreaming）。做梦通过回顾过往会话来发现模式、帮助智能体实现自我改进，从而扩展了记忆能力。我们同时也向使用托管智能体进行开发的开发者开放了结果导向（outcomes）、多智能体编排（multiagent orchestration）以及网络钩子（webhooks）。这些更新共同让智能体在极少人工引导的情况下，更有能力处理复杂任务。

### 用「做梦」构建可自我改进的智能体

做梦是一个定时进行的流程，它会回顾你的智能体会话与记忆存储，提取模式，并整理记忆，使你的智能体随时间不断改进。你可以自行决定希望保留多少控制权：做梦既可以自动更新记忆，你也可以在变更落地之前先行审阅。

做梦能够揭示单个智能体自身无法察觉的模式，包括反复出现的错误、多个智能体趋同采用的工作流，以及团队内部共享的偏好。它还会重构记忆，使其在演进过程中始终保持高信号量（high-signal）。这对于长期运行的工作以及多智能体编排尤其有用。

记忆与做梦共同构成了面向自我改进智能体的稳健记忆系统。记忆让每个智能体在工作过程中捕获其所学；做梦则在会话之间精炼这些记忆，提炼出跨智能体的共享经验，并保持记忆的时效性。

做梦已在 Claude 平台（Claude Platform）的托管智能体中提供；开发者可在此处申请访问权限。

### 交付更优的结果

借助结果导向（outcomes），你撰写一份描述「成功是什么样子」的评分标准（rubric），智能体便朝着该目标努力。一个独立的评分器（grader）会在其自身的上下文窗口中，依据你的标准评估输出，因此它不会受到智能体推理过程的影响。当某处不对时，评分器会精确指出需要改动的地方，智能体随即再做一轮。

当智能体明白「好」是什么样子时，它们能做出最好的工作。例如某种结构框架、某项演示标准，或一组需要满足的要求。有了结果导向，智能体可以对照该标准核查自己的工作并自我纠正，直到输出足够好，而无需人工审阅每一次尝试。

结果导向对于需要关注细节与详尽覆盖的任务尤其有用。它也适用于主观质量，比如文案是否符合品牌语气，或设计是否遵循视觉规范。在测试中，结果导向相较标准的提示循环（prompting loop）使任务成功率最多提升 10 分，在最难的问题上收益最大。结果导向还提升了文件生成质量：在我们的内部基准测试中，docx 任务成功率提升 +8.4%，pptx 提升 +10.1%。

你现在还可以定义一个结果，让智能体运行，并在其完成时通过网络钩子（webhook）收到通知。

### 用多个智能体处理复杂任务

当工作量过大、单个智能体难以做好时，多智能体编排（multiagent orchestration）让一个主智能体（lead agent）把任务拆分成若干部分，并把每一部分委派给一个拥有自身模型、提示词与工具的专精智能体。例如，一个主智能体可以执行一项调查，同时子智能体（subagents）分头排查部署历史、错误日志、指标与支持工单。

这些专精智能体在共享文件系统上并行工作，并贡献到主智能体的整体上下文中。由于事件是持久化的、且每个智能体都记得自己做过什么，主智能体可以在工作流进行到一半时回头与其他智能体核对。你还可以在 Claude 控制台（Claude Console）中追踪每一个步骤：哪个智能体做了什么、按什么顺序、为什么这么做，从而对你的任务如何被委派与执行拥有完整的可见性。

### 各团队正在构建什么

各团队正在使用做梦、结果导向与多智能体编排，交付能够核验自身工作、跨会话学习并并行处理复杂任务的智能体：

- Harvey 使用托管智能体来协调诸如长篇起草与文档创建等复杂法律工作。借助做梦，他们的智能体在会话之间记住所学的内容，包括文件类型的变通方法与特定工具的模式。在他们的测试中，完成率提升了约 6 倍。

- Netflix 的平台团队构建了一个分析智能体，处理来自不同来源、数百次构建的日志。对于影响数千个应用的变更，关键在于找出在众多应用中反复出现的问题。多智能体编排让该智能体能够并行分析多个批次，并仅呈现值得采取行动的模式。

- Spiral by Every 正在使用多智能体编排与结果导向，为其新推出的 API 与 CLI 背后的写作智能体提供支撑。主智能体运行在 Haiku 上：它接收来访请求，必要时提出快速的追问，然后把起草工作委派给运行在 Opus 上的子智能体。当用户要求多份草稿时，子智能体并行运行。写作质量是 Spiral 的核心价值，因此他们用结果导向来加以保证。每份草稿都依据一套评分标准被打分，该标准融合了 Every 的编辑原则与用户的语气，二者均从记忆中提取。只有达到标准的草稿才会被返回。

- Wisedocs 在托管智能体上构建了一个文档质量检查智能体，使用结果导向依据其内部准则为每一次审阅打分。审阅现在快了 50%，同时仍与团队的标准保持一致。
