# Trending Diggest

Multi-source digest archive and public reading site for technical resources.

This repository owns public reading artifacts, archive indexes, and the Jekyll
presentation contract. Source extraction, translation, scheduler configuration,
DingTalk delivery, credentials, and operational state live outside this
repository.

`article-pipeline` treats this repository as an archive destination. It renders
the same canonical article package independently for this site and for remote
publishers such as DingTalk; DingTalk documents are not generated by reading the
Markdown back from this repository.

`sources/*/state/processed.json` is temporary compatibility state for the legacy
Claude Blog producer. New producers must use private runtime state and may write
a public `sources/<source>/manifest.json` containing only published-content
lineage.

## 安装

The archive is a Jekyll site. Use the repository verification scripts and the
Ruby environment documented by the project when validating local changes.

## 使用

Readers use the generated GitHub Pages site. Content producers write through an
archive adapter that follows the contract below; this repository does not run
source extraction or translation itself.

## Producer contract

- Public posts and indexes are deterministic archive artifacts.
- Front matter strings must be valid quoted YAML scalars.
- Public manifests may contain source URLs, content hashes, archive paths, and
  generator versions, but never credentials, internal URLs, retry state, or
  delivery destinations.
- Archive producers must support a read-only dry-run and must not commit or push
  implicitly.
- Repository build validation remains authoritative for the public site.
